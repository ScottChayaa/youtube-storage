# 階段性驗收清單（Acceptance Checklist）

- **建立日期**：2026-07-05
- **用途**：每個階段/計畫完成時，照本清單逐項驗證，作為「這個階段是否達成使用者目標」的依據與紀錄。
- **怎麼用**：完成一項就把 `- [ ]` 改成 `- [x]`，並在該項的「紀錄」欄填上驗證日期與結果（PASS / FAIL / 備註）。

## 核心原則（先讀）
> **搜尋品質 = 標記品質（garbage in, garbage out）。** 系統負責「秒回、跳到正確時間點、依時間/標籤瀏覽」；但「找得到 = 先標得到」。內容語意 tag（如「貓」、貓的名字）的來源依階段不同：

| 內容 tag 來源 | 何時可用 |
|---|---|
| 資料夾名稱、日期（自動） | Plan 1 起 |
| 逐片段手動標記（時間軸編輯器） | 編輯器計畫（Plan 1–3 之後，方案 B） |
| AI 影像/人臉自動辨識 | Phase 3 |

> 依**方案 B**：在 Plan 1–3 的第一個可用版本裡，「貓」要能被搜到，需**資料夾命名**含該詞；逐片段補「貓」等內容 tag 要等時間軸編輯器計畫或 AI。
